For the longest time I used Visio to create my Entity Resource Diagrams. Then I was introduced to DBML! What Looking at a database structure as code and being able to see both the code and the visual model being generated at the same time was absolutely crazy.

While this language may be seen as just the way that a database is described, I would argue that it can also represent the way a fillable .pdf form is created or even the functionality of a user experience screen. In the end, the way that the user experience is seen may have some logic to the form, the underlying data model ends up underneath.
