The value stream networks can be modelled at three levels:
1. **Value Stream** - this high level can be used to see the big picture centered on a particular domain. This might include all value streams within a domain and the enabling value streams required for that value stream to be able to execute.
2. **Value Stream Stage** - this view is useful to understand more granular interactions between value streams inside stages for one of the domain value streams and the other enabling value streams.
3. **Capability** - this view is usually required to document how a specific business capability triggers another domain's value stream. For simple value streams networks this may include the capability mapping for all value stages.  More complicated value stream networks may need to be modelled at the value stream stage level.
